## Исходная проблема

Completeness застряла на 0.567–0.589. Error analysis показал что retrieval работает (16+ фрагментов находятся), но LLM плохо синтезирует ответ. Оригинальный промпт:

```
Ты — эксперт по нормативным документам в области охраны труда и промышленной безопасности.
На основе предоставленных фрагментов нормативных документов дай точный, структурированный ответ.
Ссылайся на конкретные документы и пункты.
Отвечай только на основе предоставленных фрагментов — не придумывай информацию.
```

## Принятые решения

### Score → категория HIGH/MED/LOW (в коде)

**Было:** `[1] (релевантность=0.82) текст`
**Стало:** `[1] (HIGH) текст`

Пороги в `make_generate_fn._score_label()`: HIGH ≥ 0.6, MED 0.4–0.6, LOW < 0.4.
Промпт: "LOW фрагменты используй только при отсутствии лучших."

Причина: число без шкалы бессмысленно для модели. Категория + правило применения — понятны.

### Противоречия: показывать, не приоритизировать

**Было:** "Более новый документ важнее старого, специальная норма важнее общей."
**Стало:** "Укажи оба источника и оба варианта. Если в тексте видны даты — укажи их. Выбор оставь пользователю."

Причина: модель не может надёжно определить иерархию норм (новый общий ≠ отменяет старый специальный если тот не отозван). Лучше явное противоречие чем неверная приоритизация.

### Дословные цитаты для ключевых формулировок

**Добавлено:** "Ключевые формулировки нормы (требования, сроки, цифры) цитируй дословно в кавычках. Остальное можно пересказывать."

Причина: при пересказе своими словами теряется точность юридических формулировок, изменяются классификации и категории.

### Три варианта ответа (полный / частичный / нет)

**Было:** "Отвечай только на основе фрагментов — не придумывай."
**Стало:** явные три пути с точными формулировками для каждого.

Вариант (b) расширен: если вопрос не содержит данных для применения нормы (вид работ, категория, должность) → дать ответ для типового сценария + явное допущение ("Допущение: [что предполагается]") + список недостающих параметров.

Причина: без явного инструктажа модель либо галлюцинирует в стиле фрагментов, либо формально говорит "не могу ответить" когда частичный ответ возможен.

### Ссылки: оба элемента обязательны

**Было:** "Ссылайся на конкретные документы и пункты."
**Стало:** "[Фрагмент N: Документ, п. X.X]" — оба элемента. Если пункта нет — "[Фрагмент N: Документ, без пункта]".

Причина: "или" провоцирует лень — модель ссылается только на номер фрагмента который пользователь не видит. Без варианта "без пункта" — выдумывает номер статьи.

### Формат Вывода

**Было:** "1-2 предложения — прямой ответ."
**Стало:** "Прямой ответ в минимальном объёме — одна фраза для да/нет, список для перечней, без преамбул и повторения вопроса."

### Первоисточники — опциональный раздел

Выводить только если источников > 2. Для коротких ответов — избыточный шум.

### Роль

**Было:** "Ты — эксперт по нормативным документам..."
**Стало:** "Ты отвечаешь СТРОГО на основе предоставленных фрагментов..."

Причина: роль "эксперта" провоцирует достраивать ответ из общих знаний. Роль "строгого читателя фрагментов" — нет.

## Финальный промпт

```
Ты отвечаешь СТРОГО на основе предоставленных фрагментов нормативных документов.
Всё что не написано явно в тексте фрагментов — не используй и не додумывай.

ПРАВИЛА (обязательны):
1. Ссылки: каждое утверждение сопровождай ссылкой [Фрагмент N: Документ, п. X.X].
   Если пункт не указан в тексте фрагмента — пиши [Фрагмент N: Документ, без пункта].
2. Цитирование: ключевые формулировки нормы (требования, сроки, цифры) цитируй дословно
   в кавычках. Остальное можно пересказывать.
3. Три варианта ответа:
   a) Полный — фрагменты покрывают вопрос: дай ответ со ссылками.
   b) Частичный — покрывают часть: ответь на покрытое, затем "Не покрыто: [что именно]."
      Если вопрос не содержит данных для применения нормы (вид работ, категория, должность) —
      дай ответ для наиболее типового сценария, явно пометив: "Допущение: [что предполагается].
      Для других сценариев ответ может отличаться." Перечисли недостающие параметры.
   c) Нет ответа — "В фрагментах ответа нет. Не хватает: [что именно]."
4. Противоречия: если фрагменты противоречат — укажи оба источника и оба варианта.
   Если в тексте видны даты — укажи их. Выбор оставь пользователю.
5. Терминология: сохраняй термины первоисточника дословно — классификации, категории,
   группы (напр. "3.3", "1Б группа", "класс В1").
6. LOW фрагменты используй только при отсутствии HIGH или MED.

ФОРМАТ ОТВЕТА:
Вывод: [прямой ответ в минимальном объёме — одна фраза для да/нет, список для перечней,
без преамбул и повторения вопроса]
Обоснование: [детали с обязательными ссылками [Фрагмент N: Документ, п. X.X]]
Первоисточники: [только если источников больше двух — список в конце; иначе пропусти раздел]
```

## Что не реализовано / открытые вопросы

- **Даты в метаданных фрагментов:** сейчас дата документа не передаётся модели. Нужно добавить `date` в metadata при индексации → показывать `[1] (HIGH, Приказ №2464н от 2021) текст`.
- **Eval с новым промптом:** не запущен (gemini-3-flash 404). Нужна рабочая модель (gemini-2.5-flash).
- **Датасет:** очищен до 36 вопросов (удалены Q34 вентиляция, Q35 ПБ склад как OOS).
